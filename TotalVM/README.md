# TotalVM Images

Distribution repository for **TotalVM** development environments.

The TotalVM macOS app reads `index.json`, then each image's `manifest.json`, and
offers the available environments for download. Images are RAW disks compressed
with zstd (`developer.raw.zst`). **No app update is needed to publish a new
image** — just add it here.

## Layout

```
totalvm-images/
  index.json                      list of available images
  developer-ubuntu/
    manifest.json                 image metadata (see below)
    icon.svg
  developer-arch/
    manifest.json
    icon.svg
  <other-image>/
    manifest.json
```

Large artifacts (`developer.raw.zst`) are attached to **GitHub Releases**, not
committed to the repo. Each manifest's `download` points at its release asset.

## index.json

```json
{
  "schema": 1,
  "images": [
    {
      "id": "developer-ubuntu",
      "name": "Total.js Developer (Ubuntu LTS)",
      "manifest": "developer-ubuntu/manifest.json",
      "official": true
    },
    {
      "id": "developer-arch",
      "name": "Total.js Developer (Arch)",
      "manifest": "developer-arch/manifest.json",
      "official": true
    }
  ]
}
```

## manifest.json

```json
{
  "id": "developer-ubuntu",
  "name": "Total.js Developer (Ubuntu LTS)",
  "version": "1.0.0",
  "os": "ubuntu",
  "arch": "arm64",
  "format": "raw",
  "compression": "zstd",
  "memory": 4096,
  "cpus": 4,
  "disk": 40,
  "download": "https://github.com/totaljs/totalvm-images/releases/download/developer-ubuntu-1.0.0/developer.raw.zst",
  "size": 0,
  "sha256": "…",
  "icon": "icon.svg",
  "description": "…",
  "author": "Total.js"
}
```

Manifests and artifacts are generated by the image builder
(`tools/image-builder` in the TotalVM project) under `dist/<id>/`.

## Publishing a new version

1. Build: `scripts/build.sh developer-ubuntu` or `scripts/build.sh developer-arch`.
2. Create a release tagged `developer-ubuntu-<version>` and attach
   `developer.raw.zst`.
3. Copy the generated `manifest.json` into `developer-ubuntu/` here.
4. Add/update the entry in `index.json`.

## Community images

Anyone can host their own repository with the same `index.json` + `manifest`
structure; the app can read any compatible repo.
